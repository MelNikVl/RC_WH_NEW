Main office IT warehouse tracker (INSTRUCTIONS)

Программа разработана для учета активов IT отдела.

Она позволяет:

  • Вести учет активов на складах
  • Следить за перемещениями активов

WEB TERMINAL (http://{адрес_сервера}/app:9000)

Создание актива (новой карточки для техники)

При создании активов вы можете выбрать несколько фото. По адресу \\fs-mo\admins\Photo_warehouse\photos\ создастся папка с ID актива и там будут храниться все фото.

Перемещение активов

При перемещении активов - каждое новое место записыватся отдельно в БД с уникальным трек номером. Так что ничего не потеряется. Так же есть кнопка - "Посмотреть перемещения актива" - где вы можете найти все места и ответственных, на ком числилась техника. Переместить можно сразу несколько активов

Удаление активов

Кнопка - "удаление". Удалить можно сразу несколько активов. При удалении стирается вся информация о технике

АВТОРИЗАЦИЯ И ДОСТУПЫ

Управление активами реализовано на основе разделения пользователей на админов и юзеров.

Токен для работы в программе действует 1 час.

Программа постоянно проверяет валидность JWT токена (он генерируется из вашего имени пользователя и пароля). Как только токен истекает - пользователя выкидывает обратно на страницу авторизации

Админы имеют все права для работы с активами и пользователями:

  • Создавать и удалять пользователей
  • Создавать и удалять активы

Пользователи могут:

  • Менять описание любого актива
  • Удалять свои активы

Чтобы стать админом, нужно:

  1. Попросить админа выдать админский доступ
  2. Авторизоваться под собой
  3. Воспользоваться эндпоинтом - become_admin (пароль 4352525)

Создание пользователей

Нов ых пользователей может создавать только администратор через веб-консоль (Swagger или приложение).

Если стал админом, нужно перелогиниться для вступления прав в силу. Потому что ты авторизован под одним JWT токеном, а для администратора он формируется другой.

Логирование

Какие действия логируются:

  • Вход в приложение
  • Создание пользователя
  • Удаление пользователя
  • Создание актива
  • Удаление актива
  • Изменение описания актива
  • Перемещение актива

Все действия пишутся в отдельную таблицу базы данных - Logs, и выводятся на веб по 100 пунктов с пагинацией.
